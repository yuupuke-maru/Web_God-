  • メディアクエリとは　2:05

    • メディアクエリとは
    サイトで表示している画面環境などを判定し、条件指定ができるＣＳＳ機能。媒体（ディスプレイ、プリンターなど）や表示デバイスの画面サイズを判別できる。


    画面サイズによって表示機能を変える機能は「メディアクエリ」というCSSの機能によって実装されている


  • 実際に指定してみよう！

    • メディアクエリを設定するには次の2つを設定。

      ①viewportの設定2:35

    htmlの「headタグ」に「」を記述

      ②条件指定　2:59

    具体的な条件の指定を「CSS」に記述する


  • メディアクエリの条件指定

    • @media screen and (min-width: 768px) { body { background-color :red; } }
    この場合媒体が「screen（一般的なディスプレイ）」で「画面サイズが768px以上」の時背景を「red(赤)」で表示

    媒体指定は「all」,「screen」「print」の３通り。
    ただし、「screen」と「print」で表示方法を変えなければいけないとき以外は記載しなくてもOK 。


    画面サイズでの条件指定

  • 768px以上の画面幅に効かせたい場合＝CSSが効く範囲の、最小値を指定

    • @media (min-width: 768px;) { h1 { font-size: 10px; } }
  • 768px以下の画面幅に効かせたい場合＝CSSが効く範囲の、最大値を指定
    • @media (max-width: 768px;) { h1 { font-size: 10px; } }
  • 768px~1199pxの範囲の画面幅に効かせたい場合＝CSSが効く範囲の、最小値が768px最大値が1199px
    • @media (min-width: 768px;) and (max-width: 1199px;) { h1 { font-size: 10px; } }

  • ブレイクポイントについて　6:54

    • 最小値や最大値に指定する画面幅の値を「ブレイクポイントという」。

  • PCとタブレットのブレイクポイントを「1200px」、タブレットとスマホのブレイクポイントを「768px」としたとき

    • スマホ表示
    div { width: 200px; }
    タブレット表示
    @media (min-width: 768px) { div { width: 400px; } }
    PC表示
    @media (min-width: 1200px;) { div { width: 600px: } }

    モバイルファースト、PCファーストがある。上位の例は「モバイルファースト設計」

  • レスポンシブコーディングをする時の手法について　10:36

      • ①パーセント指定　10:45

    img { width: 100%; }

      ②「rem」での文字サイズ指定 11:39

    PCとスマホで異なる文字サイズを設定したいときにPXでは指定箇所が多く手間がかかる。基準となる文字サイズを「htmlタグ」に指定。基準の文字サイズに対して比率でサイズを変更していく

    レスポンシブルデザインの場合はこれにメディアクエリを併用して、画面幅が一定より大きくなった場合、基準の文字サイズを大きくする設定をする

    ・スマホ表示 html { font-size: 10px; } h1 { font-size :1.2rem; } ・PC表示 @media (min-width: 768px:) { html { font-size: 20px; } }

      ③画面幅指定での要素の出し分け　13:12

    メディアクエリと「display :none;」を組み合わせて指定。

    スマホの時のみ表示させたい場合。
    ・クラス「sp_only」を用意する PC表示 @media (min-width: 768px;) { .sp_only { display :none; } }

    pcとスマホで特定の要素の表示、非表示を入れ替える ・スマホ表示 .pc_only { display :none; } ・PC表示 @media (min-width :768px;) { .pc_only { display :block; } }
