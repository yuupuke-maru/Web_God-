    • Cssが効かない時のチェックポイント

        CSSファイルが読み込めているか？

      YES/NO

      確認方法 検証ツール「Console」タブ

      CSSファイルの読み込みエラーチェック

      検証ツール→右上の赤い×印、もしくは「Console」タブを確認
      「Not Found」や「404」はファイル読み込みエラー

      検証ツール→黄色い注意マークが出ていることもあるが、これは無視してＯＫ。赤いバツ印がなければ大丈夫。

        CSSファイルが読み込めていないとき

      → CSSファイルの位置を確認。
      → CSSファイルの読み込み部分の記述を見直す。
      ・・・HTMLファイルのlinkタグ

      ケース１:CSSファイルが存在しない。
      フォルダにない？間違えて消した？サーバーにアップロード忘れ？


      ケース２:linkタグの記述ミス
      文法間違い？srcとかhrefとか、属性の混同？


      ケース3:CSSファイル名が間違っている。
      存在しないファイル名を指定していない？


      ケース4:CSSファイルの場所が違う。
      「/」や「../」を正しく使ってファイルの階層構造を表せているか？


      ケース5:linkタグの記述に、大文字が混在している文法ミス。
      「"」や「<」が、よく見たら大文字記号だった・・・なんてことも？

      閉じカッコ忘れ、セミコロン忘れ、プロパティ名間違いなど

      【W3C CSS 検証サービス】

      https://jigsaw.w3.org/css-validator/validator.html.ja#validate_by_input

      ケース2:セレクタ指定が間違っている
      半角スペースの有無や、親要素と子要素の関係性などが間違っている？


      よくある間違い

      セレクタ指定が間違っている

      h .logo { position: absolute; left :500px; } >>>>クラス「logo」が付いた「h1タグ」なので、正しくは「h1.logo」とスペースがいらない。


      header > .logo > img { width :650px; } >>>>「>」記号は直下セレクタを表す。
      この場合、「.logo」と「img」の間にaタグをはさんでいるので直下ではない。
      正しいセレクタは「header > .logo img」


      ケース3:クラス名ID名などが間違っている
      スペルミスだったり、途中で暮らす名を変更したのをわすれていたりなど

      　

      .rogo img{ width :650px; } >>>>スペルミスで存在しないクラスを指定。正しくは「.logo」


      CSSの記述順,セレクタ詳細度による上書き
      後から書いたCSSや詳細度の高いCSSに邪魔されている？

      タグ単体でのセレクタ指定は、詳細度が最も低い（うわがきされやすい） 「クラス名 img」のような指定をしているCSSが書いてあればそちらで上書きされてしまう
      リセットCSSを後から読み込んで、自分のCSSの邪魔をしている。（想定されていない上書きが起こる）


      ケース6:そもそもプロパティや、その値の使い方が間違っている。
      プロパティの使い方があっているか要確認。

      例) 文字色を変えたい

      nav li { color :red; } >>>> 中のテキストがaタグに囲まれているときはaタグに文字色指定をしないと変化しない。
      正しいセレクタは「nav li a」


      例) 見出し文字を真ん中揃えにしたいとき

      main h2 { margin: 0 auto; } >>>> 要素の中のテキストを真ん中に寄せるときは「text-align: center;」が正しい。


      検証ツールでCSSをチェック

      聞かせ値要素の「Styles」タブに、スタイル指定が表示されているか？
      ×表示されていない → セレクタ指定が間違っている
      〇表示されている → 何かに上書きされている、もしくは使い方が違う
